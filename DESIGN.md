# DESIGN.md

> Yu.A.Ye 포트폴리오 사이트의 시각 언어 정의서. 모든 UI 결정은 이 문서를 기준으로 삼는다. 추가·변경이 필요하면 이 문서를 먼저 업데이트한 뒤 코드에 반영한다.

---

## 1. 디자인 원칙

### 1.1 3개의 철칙
1. **작품이 주인공, UI는 투명하게**. 장윤영 작가 사이트처럼 프레임은 사라지고 이미지만 남아야 한다.
2. **여백은 정보다**. 빈 공간은 낭비가 아니라 "다음 작품으로 넘어가기 전에 숨 쉬는 시간"이다.
3. **한국적 감도는 색과 질감에만**. 레이아웃·타이포그래피는 서구 갤러리의 엄격한 미니멀리즘을 따르고, 한국성은 작품 이미지와 보조 색채에서만 드러낸다. 단청 패턴·전통 문양을 UI 장식으로 쓰지 않는다.

### 1.2 무드 보드
- 참조 사이트: 장윤영 작가 홈페이지, Universal Everything, Refik Anadol Studio
- 지양: 번쩍이는 그라데이션, 글래스모피즘, 과장된 hover 효과, 로딩 스피너 중심 UI
- 지향: 미술관 도록의 정밀함, 화이트큐브의 긴장감, 자개 표면의 은은한 빛

## 2. 색상 시스템

### 2.1 기본 팔레트 (Base)
이 사이트의 99%는 다음 3개 색으로 구성된다.

```css
:root {
  --ink-100: #0A0A0A;      /* 본문 텍스트, 주요 컬러 */
  --ink-60:  #525252;      /* 보조 텍스트, 메타 정보 */
  --ink-30:  #A3A3A3;      /* 비활성, 디바이더 */
  --paper:   #FAFAF7;      /* 전체 배경 — 순백이 아닌 미세한 오프화이트 */
  --paper-pure: #FFFFFF;   /* 이미지 위 오버레이 등 제한적 사용 */
}
```

> `--paper`를 순백(#FFFFFF)으로 하지 않는 이유: 작품 이미지(특히 자개의 블랙 배경)와 대비가 너무 날카로워지고 눈이 피로해진다. 미세한 웜톤 오프화이트가 한지의 감각을 준다.

### 2.2 액센트 팔레트 — Najeon (자개)
극히 제한적으로 사용. 인터랙션 상태(hover, active), 포커스 링, 페이지 전환의 일순간 등에만.

```css
:root {
  --najeon-teal:   #3FA7B8;   /* 자개의 청록 반사 */
  --najeon-violet: #8B7AB8;   /* 자개의 보랏빛 반사 */
  --najeon-amber:  #C9A961;   /* 자개의 금빛 반사 */
}
```

**사용 규칙**:
- 이 세 색을 **단독으로 사용하지 않는다**. 항상 서로 전환되는 그라데이션 형태로, hover 애니메이션의 "한순간"에만 나타난다 (200~400ms).
- 전경 텍스트 색으로 쓰지 않는다. 언더라인·작은 도트·포커스 링의 획에만.
- 링크의 기본 hover는 `--najeon-teal` 한 가지로 통일.

### 2.3 Obangsaek은 UI에 쓰지 않는다
오방색(적·청·황·백·흑)은 Al의 작품 안에서만 등장한다. 사이트 UI에 오방색을 직접 쓰면 작품과 충돌한다. 이 규칙은 강력하다.

### 2.4 다크 모드
**제공하지 않는다.** 이 사이트는 화이트큐브 미학이 정체성이다. 작품 상세 페이지 중 배경이 검정이 적합한 경우(야간 미디어 파사드 등)는 **페이지 단위로** `data-theme="dark"`를 부여해 해당 페이지만 반전시킨다.

```css
[data-theme="dark"] {
  --paper: #0A0A0A;
  --ink-100: #FAFAF7;
  --ink-60: #A3A3A3;
  --ink-30: #525252;
}
```

## 3. 타이포그래피

### 3.1 폰트 스택

| 용도 | 폰트 | 대체 |
|------|------|------|
| **영문 제목·헤드라인** | Neue Haas Grotesk Display *or* Inter Tight | system-ui |
| **영문 본문** | Inter | system-ui |
| **한글 전체** | Pretendard Variable | "Apple SD Gothic Neo", sans-serif |
| **한자** | Noto Serif KR (한자 글리프 포함) | serif |
| **특수(작품 제목 장식 시)** | Canela Deck *or* IBM Plex Serif | serif |

- **기본 지침**: 본문·네비·캡션은 전부 sans(Pretendard/Inter). serif는 작품 상세 페이지 최상단 대제목 **한 곳에만** 선택적으로 사용.
- Pretendard는 `font-weight: 300~700` 가변폰트로 로드.

### 3.2 타입 스케일 (1.250 Major Third)

```css
:root {
  --text-xs:   12px;   /* 메타, 캡션 하단 */
  --text-sm:   14px;   /* 네비, 보조 텍스트 */
  --text-base: 16px;   /* 본문 */
  --text-lg:   20px;   /* 본문 강조 */
  --text-xl:   25px;   /* 섹션 소제목 */
  --text-2xl:  32px;   /* 섹션 제목 */
  --text-3xl:  44px;   /* 페이지 제목 */
  --text-4xl:  64px;   /* 인트로 대제목 */
  --text-5xl:  88px;   /* 히어로 (극히 드물게) */
}
```

모바일에서는 `--text-3xl` 이상을 15% 축소.

### 3.3 자간·행간
- 한글 본문: `letter-spacing: -0.01em; line-height: 1.75;`
- 영문 본문: `letter-spacing: 0; line-height: 1.6;`
- 대제목(40px+): `letter-spacing: -0.02em; line-height: 1.1;`
- 네비 텍스트: 모두 **대문자**, `letter-spacing: 0.12em; font-weight: 500;` (장윤영 사이트 스타일)

### 3.4 한/영 병기 규칙
같은 정보를 한글과 영어로 병기할 때:
```
자개호소 / Najeon Tiger Roar
螺鈿虎嘯
```
- 한글을 위에, 영문을 아래 (또는 `/`로 연결).
- 한자가 있으면 별도 줄에 더 작게, `color: var(--ink-60)`.
- 영문은 한글보다 한 단계(-2px) 작은 크기 또는 `--ink-60`으로 처리하여 계층 형성.

## 4. 레이아웃

### 4.1 그리드와 거터
- **데스크톱 (1280px+)**: 12 column, gutter 32px, max-width 1440px, 양옆 padding 48px
- **태블릿 (768–1279px)**: 8 column, gutter 24px, padding 32px
- **모바일 (< 768px)**: 4 column, gutter 16px, padding 20px

### 4.2 수직 리듬
8px 그리드. 모든 margin/padding은 8의 배수에서만 선택 (`8, 16, 24, 32, 48, 64, 96, 128, 192`).

### 4.3 상단 네비게이션

```
┌────────────────────────────────────────────────────┐
│  [Yu.A.Ye]              WORKS  RECOGNITION  ABOUT  │
└────────────────────────────────────────────────────┘
    48px padding                        24px gap between items
```

- 높이: 88px 고정
- 스크롤 시 배경: `rgba(250, 250, 247, 0.85)` + `backdrop-filter: blur(12px)`
- 스크롤 위로 올라가면 네비도 등장/사라짐(hide on scroll down, show on scroll up). 프레이머 모션 `useScroll` 훅 사용.
- **인트로 페이지(`/`)에서는 네비를 숨긴다**. 로고만 좌측 상단에 작게.

### 4.4 인트로(`/`) 레이아웃

**구조**
```
┌──────────────────────────────────────┐
│ [Yu.A.Ye / 유어예]                    │   ← 좌상단 로고
│                                      │
│                                      │
│         [ 풀스크린 배경 ]              │
│      이미지 1장 또는 무음 영상 루프      │
│                                      │
│                                      │
│   WORKS   RECOGNITION   ABOUT        │   ← 하단 중앙 라벨 3개
│                                      │
│            © 2026 Yu.A.Ye            │
└──────────────────────────────────────┘
```

**배경 (풀블리드 100vh)**
- 단일 미디어 1장. 이미지(`.jpg`/`.webp`) 또는 무음 mp4 루프. 둘 다 `/public/intro/intro.{jpg,mp4}` 경로.
- 비디오 우선, 없으면 이미지, 둘 다 없으면 `--ink-100` 솔리드 폴백.
- `object-fit: cover`로 화면 채움. 비율 강제 크롭 허용 (배경 용도).
- 비디오 사양: 무음, 8~15초 자연 루프, mp4(h.264), ≤5MB, 1920×1080 권장.
- `prefers-reduced-motion: reduce` 사용자에게는 비디오 대신 포스터 이미지(`/public/intro/intro.jpg`)만 표시.

**오버레이 (가독성용 스크림)**
- 상단 0~20vh: `linear-gradient(to bottom, rgba(10,10,10,0.35), rgba(10,10,10,0))` — 좌상단 로고 가독성 확보.
- 하단 60~100vh: `linear-gradient(to top, rgba(10,10,10,0.5), rgba(10,10,10,0))` — 하단 라벨/카피라이트 가독성 확보.
- 이 두 그라데이션은 **장식이 아닌 가독성 보조**이며 §10의 "그라데이션 배경 금지" 예외다.

**라벨 (하단 중앙)**
- 3개를 가로 한 줄로 배열, 가운데 정렬. 항목 간 gap: 데스크톱 64px, 태블릿 48px, 모바일 32px.
- 각 라벨 스타일: uppercase, `text-xl`(데스크톱) / `text-lg`(모바일), `letter-spacing: 0.16em`, `font-weight: 500`, color `rgba(255,255,255,0.9)`.
- hover/focus: 색이 `--paper`(완전 흰)로 150ms 전환 + 라벨 아래 1px 흰색 언더라인이 0→100% 너비로 확장(300ms ease-out). najeon shimmer는 사용하지 않는다 (단일 배경 위에서는 색 시프트가 산만함).
- 클릭 영역은 라벨 텍스트 + 상하 좌우 12px 패딩까지.
- 모바일에서 라벨이 한 줄에 안 들어가면 줄바꿈 허용 (gap만 줄임). 세로 스택은 피한다.

**로고 (좌상단)**
- `Yu.A.Ye` 한 줄 + 그 아래 작게 `유어예`. 색은 `rgba(255,255,255,0.95)`.
- 클릭 시 `/`로 이동(자기 자신) — `<Link>`로 두되 인트로에서는 사실상 비활성처럼 동작.

**카피라이트**
- 라벨 아래 한 줄, 매우 작게(`text-xs`), color `rgba(255,255,255,0.6)`. `© {year} Yu.A.Ye`.

**모바일**
- 동일한 풀스크린 1장 + 라벨 가로 한 줄 구조 유지. hover 없는 환경이라 라벨은 기본부터 흰색 90% 투명도로 충분히 보이게.
- 라벨 폰트만 한 단계 축소(`text-lg`), 항목 간 gap 32px.

### 4.5 작품 목록(/works) 그리드
- 데스크톱 2열. **의도적으로 3~4열을 피한다** — 이미지를 크게 보여주는 것이 이 사이트의 가치.
- 카드 간 수직 간격 96px, 수평 간격 48px.
- 카드 내부: 이미지 → 16px 간격 → 제목(한) → 4px → 제목(영) → 8px → 연도·타입 메타.

### 4.6 작품 상세 페이지
- 최상단에 풀블리드(가로 100vw) 히어로 이미지/비디오. 높이는 이미지 원본 비율.
- 본문 영역은 중앙 정렬, max-width 720px. 텍스트를 좁게 유지해 가독성 확보.
- 추가 이미지는 본문 너비 또는 풀블리드를 교차 사용해 시각적 리듬을 만든다.

## 5. 컴포넌트 스펙

### 5.1 WorkCard
```
┌─────────────────────┐
│                     │
│      [thumbnail]    │   ← aspect-ratio 유지, border 없음
│                     │
└─────────────────────┘
  자개호소                 ← text-lg, ink-100
  Najeon Tiger Roar        ← text-sm, ink-60
  2026 · Projection        ← text-xs, ink-60, letter-spacing 0.08em
```

- hover: 이미지만 `scale(1.015)`, 1200ms ease-out. 텍스트는 변화 없음.
- 클릭 영역은 전체 카드. 키보드 포커스 시 카드 전체에 2px `--najeon-teal` 아웃라인 (offset 4px).

### 5.2 Button (거의 쓸 일이 없지만)
- "Load More" 버튼 정도. 기본 스타일: 배경 투명, `1px solid --ink-100`, padding `12px 32px`, `text-sm`, uppercase, tracking 0.12em.
- hover: 배경 `--ink-100`, 텍스트 `--paper`. 150ms.
- 둥근 모서리 없음 (border-radius: 0). 이 사이트는 어디에도 rounded corner를 쓰지 않는다.

### 5.3 Divider
- 1px, `--ink-30`. 섹션 구분용. 절대 그림자·그라데이션 금지.

### 5.4 링크 (인라인)
- 기본: `color: var(--ink-100)`, `border-bottom: 1px solid var(--ink-30)`.
- hover: border 색 `--najeon-teal`로 전환, 200ms.
- 외부 링크는 끝에 `↗` 붙이기 (아이콘 이미지 아님, 텍스트 문자).

## 6. 이미지 처리

### 6.1 비율
- 작품 섬네일은 **원본 비율 유지**. 강제로 정사각형/16:9 크롭 금지. 미디어 파사드의 극단적 가로비도 그대로 보여준다.
- 단, 랜딩 3분기의 대표 이미지는 모두 세로형(9:16 또는 3:4)으로 통일 — 3칼럼 레이아웃의 시각적 일관성을 위해.

### 6.2 프레임과 그림자
- 이미지 주위에 border·shadow·둥근 모서리를 **전혀 쓰지 않는다**. 이미지 그 자체가 오브제.
- 예외: 작품 상세 페이지에서 이미지 클릭 시 전체 화면 라이트박스로 열림. 라이트박스 배경은 `rgba(10, 10, 10, 0.94)`.

### 6.3 로딩
- 빌드 타임에 생성한 `blurDataURL`로 blur-up placeholder.
- 로딩 스피너를 **사용하지 않는다**. blur → 선명 전환으로 대체.

### 6.4 비디오 플레이어 (유튜브 임베드)

유튜브를 호스팅으로 쓰되 갤러리 사이트의 시각 언어를 지키기 위한 규칙. 기술적 구현은 `CLAUDE.md §5.2` 참조.

**정지 상태 (재생 전)**
- 포스터 이미지가 화면을 완전히 채움. 기본적으로 유튜브 `maxresdefault.jpg` 자동 사용 (추가 작업 없음). 특정 작품의 썸네일이 마음에 안 들면 유튜브 스튜디오에서 커스텀 썸네일을 업로드하는 쪽이 가장 간단.
- 포스터 위 중앙에 커스텀 플레이 버튼:
  ```
  ┌─────────────┐
  │             │
  │      ▷      │   ← 64px, 1.5px 흰 스트로크 원 + 내부 삼각형
  │             │
  └─────────────┘
  ```
- 플레이 버튼 색: `rgba(255, 255, 255, 0.9)`. 배경이 밝은 포스터일 때는 자동으로 `rgba(10, 10, 10, 0.8)`로 스왑 (포스터 평균 밝기 기반).
- hover: 버튼이 `--najeon-teal`로 300ms 전환, 동시에 포스터가 `brightness(1.05)`로 미세하게 밝아짐. 크기는 **변하지 않는다** (흔한 `scale(1.1)` 효과 금지 — 유튜브 느낌이 남).

**재생 중 상태**
- iframe만 표시. 주변에 Yu.A.Ye 로고, 작품 제목 등 커스텀 오버레이를 **덧붙이지 않는다** (진입 후에는 영상 자체에 집중).
- iframe은 `--paper` 배경 위에 그대로 얹힌다. 프레임·그림자·border 없음.

**종료 후 상태**
- 영상이 끝나면 iframe 언마운트 → 포스터 + 플레이 버튼으로 즉시 복귀 (150ms 페이드).
- "다시 재생" 같은 별도 라벨을 표시하지 않는다. 같은 플레이 버튼이 곧 다시 재생을 의미.

**작품 상세 페이지 내 배치**
```
┌────────────────────────────────────────┐
│                                        │

    [Statement 본문 ...]
```

- 영상 하단 메타에 **재생 시간**(`2'30"`)을 포함 (영상 작품의 관람 시간 예고는 갤러리 큐레이터 배려).
- 영상 아래 `Watch on YouTube ↗` 같은 외부 링크 버튼은 **제공하지 않는다** (유튜브로의 이탈 동선을 만들지 않는다).

**작품 목록(`/works`)의 카드**
- 카드 섬네일은 포스터 이미지 + 우하단 작은 플레이 아이콘 인디케이터(24px, 불투명도 0.7).
- 재생 시간 오버레이는 정적 작품과의 구분을 위해 우하단에 `text-xs`, `--paper` 텍스트 on `rgba(10,10,10,0.5)` 배경.
- hover 시 자동 재생되는 배경 루프는 **선택 구현**. 구현 시에는 유튜브가 아닌 자체 호스팅 5초 mp4 사용.

## 7. 인터랙션과 모션

### 7.1 기본 타이밍
```css
--ease-out: cubic-bezier(0.22, 1, 0.36, 1);
--ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);

--dur-fast:   150ms;   /* 호버 색 변경, 포커스 */
--dur-base:   300ms;   /* 일반 전환 */
--dur-slow:   600ms;   /* 레이아웃 확장, 페이지 전환 */
--dur-hero:   1200ms;  /* 이미지 스케일, 자개 반사 */
```

### 7.2 페이지 전환
- Framer Motion의 `AnimatePresence`. 기본 전환: 이전 페이지 `opacity 1 → 0` (200ms) → 새 페이지 `opacity 0 → 1` (400ms).
- 작품 상세로 진입할 때만: 섬네일이 히어로 위치로 `layoutId`를 사용해 모핑. (선택 구현, 성능 여유 시)

### 7.3 자개 반사 효과 (Najeon Shimmer)
인트로 3분기의 라벨에만 사용.

```css
@keyframes najeon-shimmer {
  0%, 100% { color: var(--ink-100); }
  33%      { color: var(--najeon-teal); }
  66%      { color: var(--najeon-violet); }
}
.shimmer-on-hover:hover {
  animation: najeon-shimmer 1.2s var(--ease-in-out);
}
```

- 한 번의 hover에 **한 번만** 재생. 무한 반복 금지.
- `@media (prefers-reduced-motion)` 시 애니메이션 제거, 색 전환도 삭제.

### 7.4 스크롤
- 기본 네이티브 스크롤 사용. 스무스 스크롤 라이브러리(Lenis 등) **쓰지 않는다**. 미술관 사이트에서 스크롤 조작은 불쾌감을 준다.
- 스크롤 기반 페이드인은 이미지에만 적용 (`IntersectionObserver`, opacity 0→1, 600ms). 텍스트는 즉시 표시.

### 7.5 커서
- 커스텀 커서 **쓰지 않는다**. 기본 시스템 커서.
- 링크/버튼 위에서만 `cursor: pointer`. 이미지 위에서 확대경 커서 같은 것도 쓰지 않는다.

## 8. 반응형 브레이크포인트

```css
--bp-sm:  640px;   /* 큰 모바일 */
--bp-md:  768px;   /* 태블릿 */
--bp-lg:  1024px;  /* 작은 데스크톱 */
--bp-xl:  1280px;  /* 표준 데스크톱 */
--bp-2xl: 1536px;  /* 큰 디스플레이 */
```

- 모바일 우선. 기본 스타일은 모바일 기준으로 쓰고 `@media (min-width)`로 확장.
- 3분기 인트로는 `--bp-md` 이하에서 세로 스택으로 전환.
- 작품 그리드는 `--bp-lg` 이상에서 2열, 그 이하 1열.

## 10. 금지 사항 (Do Not)

명시적으로 이 사이트에서 **하지 않는** 것들:

- 그라데이션 배경 (액센트 컬러 애니메이션, 인트로 가독성 스크림 §4.4 외)
- 글래스모피즘, 뉴모피즘
- 둥근 모서리 (border-radius > 0)
- 드롭 섀도우 (focus ring 제외)
- 이모지 아이콘
- 아이콘 폰트 (Font Awesome 등). 필요하면 인라인 SVG.
- 로딩 스피너
- 웹사이트 내부의 광고 배너
- 뉴스레터 구독 팝업
- 쿠키 배너를 제외한 모든 자동 팝업
- 자동 재생 사운드
- 마우스 트레일·파티클 커서
- "Scroll down ↓" 같은 스크롤 인디케이터 (인트로 포함)
- 단청·태극·한자 이미지 등을 UI 장식으로 쓰는 것 (작품 내부가 아닌 프레임에)
